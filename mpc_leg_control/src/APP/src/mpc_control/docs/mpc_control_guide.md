### 1.3 15维 MIT 命令格式

控制器发布的 `/motor_cmd` 话题为15维数组，包含3个关节各5个参数：

| 索引 | 关节 | 内容 | 说明 |
|------|------|------|------|
| 0 | 髋关节 | 目标角度 `q_d` | 固定值 `hip_angle`，不参与步态 |
| 1 | 髋关节 | 目标角速度 `qd_d` | 固定为 0 |
| 2 | 髋关节 | 前馈力矩 `τ_ff` | 固定为 0 |
| 3 | 髋关节 | 比例增益 `Kp` | `hip_kp` |
| 4 | 髋关节 | 微分增益 `Kd` | `hip_kd` |
| 5 | 大腿 | 目标角度 `q_d` | IK计算 + 零位偏置 + 滤波 |
| 6 | 大腿 | 目标角速度 `qd_d` | 雅可比映射 + 滤波 |
| 7 | 大腿 | 前馈力矩 `τ_ff` | 重力补偿 |
| 8 | 大腿 | 比例增益 `Kp` | `thigh_kp` |
| 9 | 大腿 | 微分增益 `Kd` | `thigh_kd` |
| 10 | 膝曲柄 | 目标角度 `q_d` | IK计算 + 零位偏置 + 滤波 |
| 11 | 膝曲柄 | 目标角速度 `qd_d` | 雅可比映射 + 滤波 |
| 12 | 膝曲柄 | 前馈力矩 `τ_ff` | 重力补偿（可选） |
| 13 | 膝曲柄 | 比例增益 `Kp` | `knee_crank_kp` |
| 14 | 膝曲柄 | 微分增益 `Kd` | `knee_crank_kd` |

电机端执行的力矩公式：

```
τ_output = Kp × (q_d - q_actual) + Kd × (qd_d - qd_actual) + τ_ff
```

## 2. 核心算法原理

### 2.1 坐标系与腿部模型

```
    髋关节 O ──────────── x轴 (水平，前进为正)
         │\
         │ \  L1 (大腿, 0.200m)
    y轴  │  \
   (向下) │   ● 膝关节
         │  / theta1 (膝弯角)
         │ /  L2 (小腿, 0.243m)
         │/
         ● 足端 P(x, y)
```

- **x 轴**：水平方向，前方为正
- **y 轴**：竖直方向，向下为正
- **theta2**：大腿关节角（大腿相对于竖直方向的偏转角）
- **theta1**：膝关节角（小腿相对于大腿的弯曲角，完全伸直时为0）

### 2.2 逆运动学 (Inverse Kinematics)

已知足端位置 `(x, y)`，求关节角度 `(theta2, theta1)`。

**步骤1：求膝关节角 theta1（余弦定理）**

```
L = sqrt(x² + y²)                            // 髋到足端距离
cos(π - theta1) = (L1² + L2² - L²) / (2·L1·L2)
theta1 = π - arccos(...)                      // 膝弯角
```

**步骤2：求大腿关节角 theta2**

```
phi1 = atan2(-x, y)                           // 足端方向角
phi2 = arccos((L² + L1² - L2²) / (2·L·L1))   // 大腿偏移量
theta2 = phi1 + phi2                           // 大腿总偏转角
```

> **安全处理**：`L` 被截断到 `[|L1-L2|+ε, L1+L2-ε]` 范围内，确保 `arccos` 输入不越界。

### 2.3 摆线轨迹 (Cycloid Trajectory)

摆动相采用摆线曲线，其优势是起止速度为零、加速度连续、运动平滑无冲击。

```
         高度 ▲
              │      ╭──╮     ← step_height (最高点)
              │    ╱      ╲
              │  ╱          ╲
              │╱              ╲
        ──────●────────────────●──► 水平
            起点              终点
           (τ=0)             (τ=1)
```

位置公式（τ 为归一化相位 [0,1]）：

```
x(τ) = start_x + step_length × [τ - sin(2πτ)/(2π)]
y(τ) = start_y + step_height × [1 - cos(2πτ)] / 2
```

速度公式（对时间求导，`dτ/dt = 1/period`）：

```
vx(τ) = step_length × [1 - cos(2πτ)] / period
vy(τ) = step_height × π × sin(2πτ) / period
```

### 2.4 雅可比速度映射

已知足端速度 `(vx, vy)`，通过解析雅可比矩阵映射为关节角速度：

```
[v_theta2]   [∂θ2/∂x  ∂θ2/∂y] [vx]
[v_theta1] = [∂θ1/∂x  ∂θ1/∂y] [vy]
```

各偏导数通过对 IK 公式链式求导获得，具体推导见源码 `compute_joint_velocity()` 函数的注释。

### 2.5 重力补偿前馈

为减轻 PD 控制器的负担，添加简化的重力补偿前馈力矩：

```
τ_thigh = m × g × (L1 × com_ratio) × cos(θ2)        // 大腿前馈
τ_knee  = m × g × (L2 × 0.5) × cos(θ2 - θ1)         // 膝关节前馈（可选）
```

物理含义：补偿腿部质心因重力在各关节处产生的静态力矩。

### 2.6 低通滤波

使用一阶指数移动平均（EMA）滤波器平滑关节命令，消除离散计算的高频抖动：

```
y[k] = α × x[k] + (1 - α) × y[k-1]
```

- `α` 越大 → 响应快但噪声大
- `α` 越小 → 越平滑但延迟增大
- 推荐范围：0.2 ~ 0.4

### 2.7 零位偏置

自动计算零位偏置，使关节在站立位（`x=0, y=stance_depth`）时命令角度为 0：

```
IK(0, stance_depth) → (θ2_neutral, θ1_neutral)
thigh_offset = -θ2_neutral
knee_offset  = -θ1_neutral
```

这样命令值只反映相对于站立位的偏差，便于调参。

## 3. 步态状态机

### 3.1 两相结构

每个步态周期分为**摆动相**和**支撑相**两个阶段，交替执行：

```
|<────────────── 一个完整步态周期 ──────────────>|
|                                               |
|<── 摆动相 (swing) ──>|<── 支撑相 (stance) ──>|
|  比例: swing_ratio    |  比例: 1-swing_ratio   |
|                       |                       |
| 足端抬起并向前摆动     | 足端着地并向后滑动      |
| 使用摆线轨迹           | 使用线性插值            |
```

### 3.2 摆动相轨迹

- 足端从 `-step_length/2` 经摆线曲线运动到 `+step_length/2`
- 竖直方向抬起 `step_height` 高度（摆线升余弦曲线）
- 起止速度均为零，运动平滑

### 3.3 支撑相轨迹

- 足端从 `+step_length/2` 线性滑动到 `-step_length/2`
- 竖直方向保持在 `stance_depth`（贴地）
- 模拟地面支撑时足端相对髋关节的运动

### 3.4 步态切换

通过 `/gait_mode` 话题切换步态时：

1. 加载目标步态的参数集
2. 重新计算摆动/支撑相时长
3. 从支撑相 50% 处重启状态机（避免跳变）
4. 重新计算零位偏置

## 4. 参数详解

参数文件：`config/mpc_gait_controller.yaml`

### 4.1 机器人物理参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `link_length_thigh` | double | 0.200 | 大腿连杆长度 [m]，必须与实际机构匹配 |
| `link_length_shank` | double | 0.243 | 小腿连杆长度 [m]，必须与实际机构匹配 |
| `leg_mass` | double | 1.05 | 腿部等效质量 [kg]，用于重力补偿计算 |
| `leg_com_ratio` | double | 0.45 | 质心在大腿上的位置比例 [0~1]，0=髋端，1=膝端 |
| `enable_knee_gravity_comp` | bool | true | 是否为膝关节添加重力补偿前馈 |

### 4.2 步态参数（walk / run / jump）

| 参数 | 说明 | walk | run | jump |
|------|------|------|-----|------|
| `step_length` | 步长 [m] | 0.08 | 0.12 | 0.00 |
| `step_height` | 抬腿高度 [m] | 0.05 | 0.06 | 0.12 |
| `period` | 步态周期 [s] | 0.80 | 0.50 | 0.65 |
| `swing_ratio` | 摆动相占比 | 0.60 | 0.60 | 0.45 |
| `stance_depth` | 站立深度 [m] | 0.29 | 0.28 | 0.30 |

### 4.3 PD 增益

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `hip_angle` | 3.21 | 髋关节固定目标角度 [rad] |
| `hip_kp` / `hip_kd` | 3.0 / 0.8 | 髋关节 PD 增益 |
| `thigh_kp` / `thigh_kd` | 12.0 / 4.0 | 大腿关节 PD 增益 |
| `knee_crank_kp` / `knee_crank_kd` | 12.0 / 4.0 | 膝曲柄关节 PD 增益 |

### 4.4 其他参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `control_rate` | 200.0 | 控制循环频率 [Hz] |
| `gait_mode` | "walk" | 启动时的默认步态模式 |
| `auto_zero_offsets` | true | 是否自动计算零位偏置 |
| `thigh_offset` | 0.0 | 大腿手动零位偏置 [rad]（auto=false时生效） |
| `knee_offset` | 0.0 | 膝关节手动零位偏置 [rad]（auto=false时生效） |
| `filter_alpha` | 0.25 | 低通滤波系数 [0.01~1.0]，越小越平滑 |

### 4.5 预留 MPC 参数（当前版本未使用）

以下参数已在 yaml 中声明，预留给后续 MPC 优化器升级：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `horizon_steps` | 12 | MPC 预测步数 |
| `horizon_dt` | 0.02 | MPC 预测步长 [s] |
| `q_pos` | 120.0 | 位置跟踪权重 |
| `q_vel` | 8.0 | 速度跟踪权重 |
| `r_acc` | 1.0 | 加速度惩罚权重 |
| `acc_limit` | 40.0 | 加速度约束 [m/s²] |
| `observer_gain` | 0.15 | 状态观测器增益 |

## 5. 使用方法

### 5.1 编译

```bash
source /opt/ros/humble/setup.bash
cd /home/llq/mpc_leg_control
colcon build --packages-select mpc_control
source install/setup.bash
```

### 5.2 启动仿真 + 控制器

**终端1** — 启动 Gazebo 仿真：

```bash
source /opt/ros/humble/setup.bash
cd /home/llq/mpc_leg_control && source install/setup.bash
ros2 launch leg_model gazebo.launch.py
```

**终端2** — 启动步态控制器：

```bash
source /opt/ros/humble/setup.bash
cd /home/llq/mpc_leg_control && source install/setup.bash
ros2 launch mpc_control mpc_gait_sim.launch.py
```

### 5.3 切换步态

```bash
# walk 步态
ros2 topic pub /gait_mode std_msgs/msg/String "{data: walk}" -1

# run 步态
ros2 topic pub /gait_mode std_msgs/msg/String "{data: run}" -1

# jump 步态
ros2 topic pub /gait_mode std_msgs/msg/String "{data: jump}" -1
```

### 5.4 使用自定义参数文件

```bash
ros2 launch mpc_control mpc_gait_sim.launch.py \
    param_file:=/absolute/path/to/your_params.yaml
```

## 6. 调参指南

### 6.1 推荐调参顺序

```
1) 确认连杆长度 → 2) walk 步态调稳 → 3) 调整 PD 增益 → 4) 加速步态
```

### 6.2 常见问题与解决方案

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 关节剧烈抖动 | Kp 过大 或 滤波不足 | 降低 `thigh_kp`/`knee_crank_kp`，或减小 `filter_alpha` |
| 关节响应迟钝 | Kp 过小 或 滤波过强 | 适当增大 Kp，或增大 `filter_alpha` |
| 腿部下垂无力 | 重力补偿不足 | 检查 `leg_mass`、`leg_com_ratio` 是否准确 |
| 步态不稳定 | 步长过大或周期过短 | 减小 `step_length`，增大 `period` |
| 摆动相卡顿 | 步高太低或太高 | 调整 `step_height` 至合适范围 |
| 切换步态时跳变 | 正常（状态机重置） | 已有从支撑相中段启动的缓冲机制 |

### 6.3 调参技巧

1. **先稳后快**：先用 walk 步态（长周期、短步长）确认稳定，再逐步缩短周期、增大步长
2. **PD 增益**：先调 Kp 获得足够跟踪精度，再调 Kd 抑制振荡（Kd 一般为 Kp 的 1/3）
3. **滤波系数**：`filter_alpha = 0.2~0.3` 通常是较好的平衡点
4. **站立深度**：`stance_depth` 决定腿部伸展程度，接近 `L1+L2` 时腿几乎伸直（奇异位置），建议保持在 `0.7×(L1+L2)` 左右
5. **跳跃调参**：先调 `step_height` 和 `period`，`step_length` 保持为 0（原地跳）

## 7. 当前实现边界与未来演进

### 7.1 当前实现

当前版本为轻量化工程实现，主要目标是快速接入仿真并验证步态：

- ✅ 足端轨迹规划（摆线 + 线性）
- ✅ 二连杆逆运动学
- ✅ 解析雅可比速度前馈
- ✅ 简化重力补偿
- ✅ 三步态在线切换
- ✅ MIT 命令接口输出

### 7.2 未覆盖内容

- ❌ 完整 MPC 滚动优化求解器（yaml 中 MPC 参数已预留，代码待实现）
- ❌ 完整刚体动力学约束
- ❌ 接触力优化与地面反作用力估计
- ❌ 全身控制器 (WBC) 协同

### 7.3 后续演进路径

如需逐步接近完整 NMPC+WBC 框架，可在当前基础上：

1. **引入 MPC 求解器**：使用预留的 MPC 参数，构建离散二阶模型滚动优化（QP 求解器如 qpOASES）
2. **更高维状态建模**：扩展到机身姿态、关节实际反馈的闭环状态估计
3. **接触时序约束**：引入接触调度器，根据地面反作用力约束优化力分配
4. **全身协同控制**：与 WBC 层级式架构集成，实现全身运动规划与执行
